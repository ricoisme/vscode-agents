# learn-topic 詳細工作流程

本文件提供 `learn-topic` skill 的完整執行參考，包含六步驟詳細說明、輸出格式模板與品質標準。

---

## 步驟 1：釐清需求

1. 確認四項輸入資訊皆已提供（學習主題、目的、期望效果、等級）。
2. 若任一欄位缺失，逐一詢問，**不得跳過或自行假設**。
3. 若等級描述模糊（如「我有一點基礎」），詢問判斷問題：
   - 「你之前有實際用過 [主題] 嗎？用在什麼情境？」
   - 「你是否寫過與 [主題] 相關的程式碼並部署到真實環境？」
4. 根據目的與期望效果，校準學習深度（廣度 vs 深度）：
   - 轉職目的 → 偏廣度，涵蓋面試常見主題
   - 專案應用 → 偏深度，聚焦直接可用的技能
   - 個人興趣 → 依使用者偏好調整

## 步驟 2：產生學習 Roadmap

依等級提供差異化的階段式路徑，每階段須標明：**目標、預計產出、建議練習時間**。

### 入門路徑
1. **環境建置與概念入門** — 安裝工具、理解核心術語
2. **基本操作** — 跑通官方 Getting Started
3. **第一個完整範例** — 從零到一完成最小可用功能
4. **常見使用情境** — 了解日常最頻繁的 2-3 個應用場景

### 中階路徑
1. **盤點已知概念** — 找出薄弱環節
2. **深化核心機制** — 理解為什麼，不只是怎麼做
3. **整合實際應用場景** — 連結工作或專案需求
4. **工具鏈整合** — 搭配 CI/CD、測試、監控等周邊工具

### 高階路徑
1. **系統化架構觀** — 從全域視角理解設計決策
2. **效能與安全考量** — 識別瓶頸與漏洞
3. **最佳實踐深探** — 業界標準與常見反模式
4. **複雜場景應對** — 多服務整合、大流量、故障排查

### 進階路徑
1. **底層原理與源碼閱讀** — 理解核心機制的實作
2. **架構設計決策分析** — 比較不同方案的取捨
3. **邊緣案例與已知問題** — 重現並分析真實 Bug
4. **社群前沿動態** — RFC、重大版本變更、社群討論

### 實戰路徑
1. **真實專案場景分析** — 從需求到架構的全流程
2. **問題診斷策略** — 有系統地縮小問題範圍
3. **生產環境注意事項** — 可觀測性、回滾策略、容量規劃
4. **團隊協作規範** — Code Review、文件、知識共享

## 步驟 3：核心概念清單

列出 5-10 個必須掌握的概念，每個概念需包含：
- **名稱**
- **一句話說明用途**
- **優先順序標記**

| 標記 | 意義 |
|------|------|
| 🧱 地基 | 必須先懂，其他概念依賴它 |
| 🔧 核心應用 | 日常開發最常用 |
| 🌿 延伸 | 進階場景才會用到，有需要再深入 |

**排序原則**：先列地基，再列核心應用，最後延伸。

## 步驟 4：重點提醒與常見陷阱

### 初學者常犯錯誤（列 3-5 個）

每項格式：
```
⚠️ **陷阱**：[具體錯誤行為描述]
   → **正確做法**：[應該怎麼做，附說明理由]
```

### 業界常見誤區（列 2-3 個）

每項格式：
```
💡 **最佳實踐**：[建議描述]
   背景：[為什麼這是誤區，以前怎麼做，現在的正確方向]
```

**注意**：陷阱必須基於真實常見錯誤，不得泛泛而談（如「要仔細閱讀文件」這種無用建議）。

---

## 步驟 5：實戰練習任務

### 任務設計原則

- 每個任務需有明確的**起點**（能從哪裡開始）與**終點**（怎樣算完成）
- 任務難度對應使用者等級，不過難也不過易
- 強調動手做，避免只是「讀完某份文件」

### 任務格式

```
### 任務：[任務名稱]

**任務描述**：
[2-3 句話說明要做什麼，情境背景是什麼]

**涉及概念**：
- [概念 1]
- [概念 2]
- [概念 3]

**驗收標準**：
- [ ] [可量化的完成條件 1]
- [ ] [可量化的完成條件 2]
- [ ] [可量化的完成條件 3]

**參考提示**（選填）：
[若有卡關點，給一個方向性提示，不直接給答案]
```

### 各等級練習目標

| 等級 | 練習目標 | 複雜度 |
|------|----------|--------|
| 入門 | 完成官方 Getting Started，跑出第一個結果 | 低：跟著步驟走 |
| 中階 | 建立一個小型但完整的功能模組，涵蓋核心流程 | 中：需要整合多個概念 |
| 高階 | 設計並實作涵蓋多個概念的完整子系統 | 高：需要架構思考 |
| 進階 | 重現一個已知問題並從源碼層級分析解決 | 高：需要閱讀源碼 |
| 實戰 | 模擬真實工作場景的完整任務（含測試、部署、監控） | 很高：端到端流程 |

## 步驟 6：精選資源清單

### 資源篩選標準

- ✅ 廣受社群認可（高星數、高評分、頻繁被引用）
- ✅ 持續維護更新（近 1 年內有更新）
- ✅ 與使用者等級相符
- ❌ 排除：過時版本、冷門資源、付費牆資源（除非確實最佳）

### 各類別精選數量

| 類別 | 精選數 | 備註 |
|------|--------|------|
| 📖 官方文件 | 1 | 標注最關鍵的 2-3 個章節 |
| 🎥 影片教學 | 1-2 | 優先推薦完整系列，非零散片段 |
| 📚 書籍 | 0-1 | 僅推薦公認經典，必須說明為何值得讀 |
| 🛠 互動練習平台 | 1-2 | 可邊做邊學，如 Katacoda、Play with Docker |
| 🌐 社群資源 | 1-2 | 官方論壇、活躍的 Discord、Stack Overflow 標籤 |

## 品質自我檢核清單

在輸出前，確認以下項目：

- [ ] 學習計畫確實對應使用者的等級，非通用模板
- [ ] Roadmap 每個階段都有明確的目標和可交付產出
- [ ] 核心概念正確標注地基/核心應用/延伸
- [ ] 至少包含 3 個具體、真實的常見陷阱（非泛泛建議）
- [ ] 至少 1 個業界誤區或過時做法被點出
- [ ] 實戰任務有明確的驗收標準（可量化）
- [ ] 推薦的資源確實廣受認可且近期有更新
- [ ] 建議時程符合該等級的實際學習曲線，不過分樂觀
- [ ] 整體建議中，實作相關內容佔比 ≥ 40%
- [ ] 輸出檔案名稱為 learn-{{學習主題}}.md
